# AutoMakeosuFile 项目结构规范

## 目录结构
1. **automakeosufile/**: 核心算法代码
   - audio_processing.py: 音频处理模块
   - feature_extraction.py: 特征提取模块
   - beatmap_generator.py: 谱面生成模块
   - config.py: 配置参数
   - main.py: 主程序入口

2. **output/**: 输出目录
   - 生成的谱面文件 (.osu)
   - 处理结果图片

3. **temp/**: 临时分析脚本和临时数据
   - 所有临时文件必须添加时间戳
   - 临时文件应定期清理

4. **picture/**: 图片保存目录
   - 所有生成的图片自动保存到此目录
   - 不需要弹出界面显示图片

5. **audio/**: 音频文件目录
   - 输入音频文件 (mp3/wav)
   - 部分osu文件

6. **docs**: 文档
   - 项目蓝图
   - 企划
   - 相关研究
   - 代码工程修改日志

## 重要指导原则

### 1. 谱面生成流程
- 只处理音频的前20秒片段进行算法测试
- 完整谱面生成由用户手动触发
- 生成的.osu文件必须自动复制到osu!歌曲目录
- 不需要在界面中显示完整谱面分析

### 2. 文件处理规范
- 命名规范: 临时文件必须添加时间戳: `filename_YYYYMMDD_HHMMSS.ext`
- 图片自动保存到picture目录，不弹出plt.show()显示窗口
- 输出文件保存到output目录
- 临时文件保存到temp目录

### 3. OSU文件处理
- 生成的.osu文件自动添加到osu!路径
- 默认osu!路径: `D:\osu!\Songs\`
- 每个谱面创建独立文件夹: `歌曲名_automake/`
- 包含音频文件和谱面文件

### 4. 算法优化原则
- 优先保证谱面的可玩性
- 避免音符密度过大
- 保持轨道分布相对均匀
- 考虑物理手感（手指移动距离）

### 5. 用户交互
- 命令行界面为主
- 进度信息输出到控制台
- 错误信息明确提示
- 不依赖图形界面交互

## 配置参数
- 默认键数: 7K
- 默认处理时长: 20秒（测试模式）
- 密度控制阈值: 动态调整
- BPM检测: 使用librosa库

## 版本控制
- 主要代码在automakeosufile目录
- 实验性代码在temp目录
- 输出文件不提交到版本控制
- 图片文件保存到picture目录
