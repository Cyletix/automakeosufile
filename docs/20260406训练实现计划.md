# 训练实现计划（先把基础打稳，再开始学）

## 目标

训练目标不是“直接从音频端到端吐出谱面”，而是：

1. 先把音频和 GT 谱面稳定对齐
2. 先验证特征是否可靠
3. 再用这些特征训练一个可解释、可回溯的映射模型

---

## 一、总体路线

建议按五个阶段推进：

### 阶段 1：数据集入口标准化

**目标：把 `D:/osu!/Songs` 变成可遍历的数据集。**

需要做的事：

1. 扫描所有歌曲目录
2. 读取每个 `.osu` 的 `[General] -> AudioFilename`
3. 把 `.osu` 和同目录音频文件配对
4. 仅保留 `Mode: 3` 的 `osu!mania` 谱面
5. 记录：
   - song folder
   - audio path
   - osu path
   - key count (`CircleSize`)
   - difficulty/version
   - creator
   - title / artist

建议产物：

- 一个 `dataset_index.csv` 或 `dataset_index.parquet`
- 一个统一的数据加载器

### 阶段 2：高质量特征提取接口

**目标：统一产出可复用特征，而不是每次在脚本里临时算。**

第一版先固定这几类：

1. **STFT complex**
   - 后续做 masking / iSTFT 必须保留复数谱和相位

2. **STFT magnitude / power / dB**
   - 供可视化、能量分析、NMF 使用

3. **Mel spectrogram**
   - 用于能量分布和后续模型输入

4. **Energy / RMSE / Spectral Flux / Onset Envelope**
   - 用于起音检测和密度控制

5. **Onset candidates**
   - 不是最终 note，只是候选事件

6. **CQT**
   - 必须直接从原始音频计算，不从 STFT 转

7. **Chroma from CQT**
   - 用于音高类 / 轨道映射分析

8. **可选：HPSS（谐波/打击乐分离）**
   - 后续可辅助 onset 与 chroma 质量提升

建议产物：

- `features.npz` 或分项缓存
- 每个样本一份 `feature_meta.json`

### 阶段 3：特征质检体系

**目标：先证明特征靠谱，再允许进入训练。**

这个阶段是当前最重要的。

#### 3.1 Onset 质检

做两个输出：

1. **click-track 混音试听**
   - 把检测到的 onset 时间打成 click
   - 和原曲叠加导出

2. **onset vs GT 时间图**
   - 横轴时间
   - 画算法 onset 和 GT note time

检查指标：

- Precision
- Recall
- F1
- 平均绝对时间偏差

#### 3.2 BPM / Grid 质检

做两个输出：

1. **grid click 试听**
   - 播放全曲节拍器
   - 至少检查开头与结尾是否同时对齐

2. **GT note 对网格的偏差分布**
   - 看 GT notes 离最近网格线的误差直方图

检查指标：

- 网格吸附误差中位数
- 95% 误差分位数
- 不同细分（1/4, 1/8, 1/16）覆盖率

#### 3.3 Chroma / 音高类质检

做两个输出：

1. **Chroma 热力图叠加 GT note**
2. **时间平滑前后对比图**

检查要点：

- 谱面主旋律区是否有明显亮带
- 打击乐是否严重污染 chroma
- 是否需要先做 HPSS / 人声或旋律增强

#### 3.4 NMF / 目标音色质检

这一阶段可以晚一点做，但一旦开始就必须有：

1. 目标成分音频导出
2. residual 音频导出
3. 分量激活矩阵可视化

检查方式：

- 直接听
- 看目标成分是否时间连续
- 看 residual 是否仍残留大量目标音色

## 二、GT 标签设计

训练前必须把 `.osu` 解析成结构化标签。

对 `osu!mania` 来说，至少需要：

1. `time_ms`
2. `lane`
3. `is_hold`
4. `end_time_ms`（若为长条）
5. `key_count`
6. `bpm_segments` / timing points
7. `audio_offset` / inherited timing 影响项

建议做两套标签：

### 1. 事件级标签

用于评估和后处理：

- 一个 note 一条记录

### 2. 帧级标签

用于训练：

- 每个时间帧对应一个状态
- 例如：
  - 是否有 onset
  - 每个 lane 是否有 tap
  - 每个 lane 是否处于 hold 中

这样模型更容易和 Mel/Chroma 等逐帧特征对齐。

## 三、先做规则基线，不直接端到端

当前建议先做一个强规则 baseline，作为训练前对照组。

### Baseline A：节奏基线

输入：

- onset envelope
- energy
- beat grid

输出：

- 候选 note 时间

规则：

- 峰值检测
- 能量门限
- 吸附到最近网格
- 密度上限控制

### Baseline B：轨道映射基线

输入：

- chroma
- spectral centroid
- 局部能量重心

输出：

- lane 分配

规则：

- 音高低 -> 左轨
- 音高高 -> 右轨
- 连续音平滑避免左右乱跳

### Baseline C：长条基线

输入：

- 持续激活的谐波成分
- 平滑后的能量包络

输出：

- hold 起止时间

规则：

- 同一 lane 上相邻短间隔事件合并
- 长时稳定能量触发 hold

只要规则基线都做不到“勉强能玩”，就说明不是模型问题，而是特征或标签问题。

## 四、训练应该怎么开始

### 推荐的第一版训练任务

不要一上来做整张谱面的 seq2seq，而是拆成三个子任务：

1. **Onset / note-time 分类**
   - 每帧判断是否应生成 note

2. **Lane 分类**
   - 在 note 已知的前提下，判断该落在哪条轨道

3. **Hold 检测**
   - 判断该 note 是否为长条，以及持续多久

这样每个错误都能追踪到具体分支。

### 推荐的第一版模型

先轻量、可解释：

1. `XGBoost / RandomForest`
   - 用于验证特征是否有区分度

2. `MLP / 1D CNN / BiLSTM`
   - 在逐帧特征上做时序建模

3. 最后才考虑 Transformer 或端到端更复杂结构

当前不建议直接上超大模型，原因是：

- 数据脏时更容易过拟合错误模式
- 不利于定位是 onset、grid、lane 还是 hold 出问题

## 五、训练评估指标

至少要拆成四组：

### 1. 时间指标

- onset precision / recall / F1
- 时间偏差 MAE
- 吸附后误差分布

### 2. 轨道指标

- lane accuracy
- top-k lane accuracy（可选）
- 轨道分布 KL divergence（看是否偏科）

### 3. 长条指标

- hold precision / recall / F1
- 持续时长 MAE

### 4. 可玩性指标

- 同轨最小间隔违规率
- 局部 NPS 峰值
- 轨道不平衡程度
- 连续跳手代价

这样你才知道该调哪个模块。

## 六、怎么定位效果不好时的问题

建议固定一个排查顺序，不要乱调：

1. **先看音频和 GT 是否对齐**
   - 如果 offset 错了，后面全错

2. **再看 onset F1**
   - 如果时间点大面积错，先不要碰 lane 模型

3. **再看 grid snap 误差**
   - 如果节拍吸附差，说明 BPM/offset 还有问题

4. **再看 lane accuracy**
   - 如果时间对了但轨道乱，问题在 Chroma / 映射逻辑

5. **最后看 hold 质量**
   - 长条是最容易被上游误差污染的

简化原则：

- **时间不对，先别调轨道。**
- **轨道不对，先别调长条。**
- **特征不稳，先别训大模型。**

## 八、现在最建议你立刻做的事情

如果只做一件最关键的事，我建议是：

## **先实现“单曲调试闭环”。**

闭环定义如下：

1. 输入一首歌目录
2. 自动找到音频和 `.osu`
3. 解析 GT note
4. 提取 STFT / Mel / Energy / Onset / CQT / Chroma
5. 导出质检图和试听文件
6. 计算 onset/grid/lane 基础指标

为什么先做这个？

因为只要单曲闭环打通，你就能非常快地回答：

- 特征到底行不行
- parser 对不对
- offset 有没有歪
- 当前最该修 onset、grid 还是 lane

这一步完成后，再扩展到批量训练，成功率会高很多。

---

## 九、最后的建议

你当前最缺的不是“再想一个更高级的模型”，而是：

1. 可复现的数据入口
2. 可验证的特征质量
3. 可量化的 GT 对齐指标
4. 可回放的调试闭环

所以真正的起点应该是：

**先把单曲闭环做成，再把它批量化，最后才开始训练。**
