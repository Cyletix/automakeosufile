## 这次整理做了什么

### 1. 收口顶层入口

- `main.py` 不再承载历史上那一大段实验代码
- 现在它只负责进入新的单曲检查流程

### 2. 收口旧入口

- `main_old.py` 已退役
- 仍保留文件名，避免历史引用或搜索路径突然失效

### 3. 新建第一版规范骨架

- `automakeosufile/dataset/song_index.py`
  - 扫描 `D:/osu!/Songs`
  - 过滤并索引 `osu!mania` 样本

- `automakeosufile/parsers/osu_mania.py`
  - 解析 `.osu`
  - 提取 `Mode`、`CircleSize`、`TimingPoints`、`HitObjects`
  - 支持 mania hold note 的结束时间解析

- `automakeosufile/features/extractor.py`
  - 统一提取 STFT、Mel、RMS、Onset、CQT、Chroma

- `automakeosufile/tools/inspect_sample.py`
  - 单曲检查入口
  - 用于后续特征质检与对齐验证的起点

### 4. 新增 evaluation 层

- `automakeosufile/evaluation/metrics.py`
  - 计算 onset 对齐指标
  - 计算 GT note 的 beat grid 吸附误差

- `automakeosufile/evaluation/visualization.py`
  - 导出固定 10 秒窗口的长图式 onset overlay 图
  - 导出固定 10 秒窗口的长图式 chroma overlay 图

- `automakeosufile/evaluation/audio.py`
  - 导出 GT note 与检测 onset 叠加的 click-track 音频

- `automakeosufile/evaluation/metrics.py`
  - 新增 lane 分布、和弦规模、hold 比例与持续时长指标

### 6. 修复一次关键可视化 Bug

- `onset_overlay` 底部的 GT note 之前把普通 note 画成了白色
- 由于图像背景默认是浅色，普通 tap note 肉眼几乎不可见
- 这会造成“Onset 图看起来 GT 很稀疏，而 Chroma 图很密”的假象
- 现在已改为：tap 使用金色、hold 使用红色，并显式设置暗色背景

这个问题说明：

**质检工具本身也必须被质检。**

## 当前一次实际调参结果（Aragami 样本）

在 `xi - Aragami` 这个 6K 高密度样本上，做了两组 onset 参数对比：

### 默认参数

- Onsets: 498
- Precision: 0.849
- Recall: 0.186
- F1: 0.305

### 高密度参数预设

- aggregate = `mean`
- delta = `0.02`
- pre_max / post_max / pre_avg / post_avg = `1`
- wait = `1`

结果：

- Onsets: 872
- Precision: 0.905
- Recall: 0.347
- F1: 0.502

### 当前结论

- 高密度 preset 明显改善了漏报问题
- 当前短板仍然主要是 Recall 偏低，而不是 Grid 偏差
- 下一步应该继续围绕 onset / energy / 特征覆盖做定向调参，而不是提前进入生成阶段

### 5. 彻底清退旧 parser / generator

- `fileprocess/osu_file_parse.py` 已改为废弃入口
- `fileprocess/osu_file_make.py` 已改为废弃入口
- 这样可以避免旧逻辑继续误导到非 mania 或错误解析流程
